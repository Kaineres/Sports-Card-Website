# Frontend Framework Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a Turborepo monorepo with a Next.js 16 web app and a shared TypeScript package, without changing what Vercel currently deploys (`index.html` stays live and byte-identical).

**Architecture:** Two-zone repo. Zone A is the existing static deploy (`index.html` + root files Vercel ships today). Zone B is the new monorepo (`apps/`, `packages/`) which only runs locally. A `vercel.json` + `.vercelignore` pair pins Vercel to Zone A so adding Zone B cannot disturb the live site.

**Tech Stack:** Turborepo, pnpm 11 workspaces, Next.js 16 (App Router, Turbopack, TypeScript), Tailwind CSS v4, shadcn/ui, Prettier.

## File Plan

**New files at repo root:**
- `pnpm-workspace.yaml` — declares `apps/*` and `packages/*` workspaces.
- `package.json` — root dev-deps (turbo, typescript, prettier), workspace scripts.
- `turbo.json` — Turborepo pipeline (dev, build, lint, typecheck).
- `vercel.json` — pins Vercel to no-build, serve-static behavior.
- `.vercelignore` — excludes monorepo files from Vercel uploads.
- `.prettierrc` — Prettier config.

**New directories:**
- `apps/web/` — Next.js 16 app (generated by `create-next-app`).
- `packages/shared/` — empty TS package skeleton.

**Modified files:**
- `.gitignore` — append `node_modules/`, `.turbo/`, `.next/`, `dist/`, `apps/*/.next/`.

**Unchanged files (must not be touched):**
- `index.html`
- `CLAUDE.md`
- `docs/`
- `supabase/`
- `.vercel/`

## Task Order

All tasks run on a single branch `scaffold/nextjs-monorepo`. Tasks 1–6 each end with a commit. Task 7 is a local verification gate. Task 8 is the Vercel preview byte-diff gate. Task 9 opens the PR.

## Rollback Notes

- **Pre-merge (anywhere in Tasks 0–8):** delete the branch. `main` and production are untouched.
  ```bash
  git switch main
  git branch -D scaffold/nextjs-monorepo
  git push origin --delete scaffold/nextjs-monorepo  # only if already pushed
  ```
- **Post-merge (would only happen if the user merged despite a failing Task 8 gate):** revert the merge commit on `main`. Vercel redeploys the prior state. Live site returns to `index.html`-only.
